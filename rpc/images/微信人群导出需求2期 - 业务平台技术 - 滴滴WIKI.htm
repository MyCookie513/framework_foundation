微信人群导出需求2期

转至元数据结尾
  • 由 李岩创建, 最后修改于2分钟以前
转至元数据起始

一、背景&目标

1期实现了微信人群的采集（维度包含app_id、template_id、用户授权数），人群基于上述维度的筛选导出。

2期需要打通标签系统，借助标签系统人群已有的其他维度进一步的筛选人群。

二、方案设计

  1. 首先要解决的是open_id和pid/did的映射问题 - 微信人群使用的是open_id，而大数据标签系统提供的人群是did或pid
    1. 黄翔会提供该映射的hive表

type CalTree struct {

type CollectUnit struct {
   SourceType int `json:"source_type"`

}

type CalUnit struct {
   CalType     int `json:"cal_type"`
